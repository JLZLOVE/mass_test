## 一、概述

`ErrorConfig` 类集中管理项目中的业务异常码及错误信息，采用 `public static final` 常量定义，便于全局引用和统一维护。

**编码设计原则**：

- 按业务领域划分编码段，便于快速定位问题来源。
- 通用 HTTP 状态码（如 400）保持标准含义，不与业务码混淆。
- 常量命名采用 `{模块}_{描述}_CODE` 和 `{模块}_{描述}_MSG` 格式，语义清晰。

------

## 二、编码区间分配

| 编码段 | 模块       | 说明                                        |
| :----- | :--------- | :------------------------------------------ |
| `400`  | 通用/HTTP  | 标准请求参数无效                            |
| `1xxx` | 用户模块   | 注册、登录、用户名/密码校验、用户删除权限等 |
| `6xxx` | 活动模块   | 活动分类、内容、编号校验                    |
| `7xxx` | 通用校验   | 非法字符、格式校验等（预留）                |
| `8xxx` | 认证/Token | 登录态、Token 解析/过期/缺失                |
| `9xxx` | 角色权限   | 角色 CRUD、等级校验、禁用、分配冲突等       |
| `91xx` | 菜单管理   | 菜单增删改查、循环依赖、组件校验等          |

| :------------------- | :----- | :----------------- |

| `TOKEN_EXPIRED_CODE` | `8003` | token已过期        |
| `TOKEN_MISSING`      | `8004` | 请求中未携带token  |

> **⚠️ 重复定义提醒**：
> 原代码中还有 `NOT_LOGGED_IN_CODE` 和 `TOKEN_FLASE_CODE` 与上述 `8001`、`8002` 含义完全一致，已由用户在最终版本中移除。**请确保使用此表中的常量，避免混用**。
