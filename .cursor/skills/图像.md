# 门户图像上传设计

> 状态：**已实现**（2026-07-18）  
> 代码变更时同步更新本文档与 `PROJECT_OVERVIEW.md`。

## 1. Controller 设计

### 1.1 PortalAdminController（已实现）

路径 `/portal/admin`，类级别 `@RequiresLevel(minLevel = Level.ADMIN)`。

| 方法 | URL | 参数 | 说明 |
|---|---|---|---|
| `uploadNoticeCover` | `POST /portal/admin/upload/notice-cover` | `@RequestParam MultipartFile file`, `@RequestParam String noticeNo` | 上传通知封面图 |
| `uploadActivityCover` | `POST /portal/admin/upload/activity-cover` | `@RequestParam MultipartFile file`, `@RequestParam String activityNo` | 上传活动封面图 |
| `uploadClubLogo` | `POST /portal/admin/upload/club-logo` | `@RequestParam MultipartFile file`, `@RequestParam Long clubId` | 上传社团 Logo |

公开门户 `/portal`（`@IgnoreAuth`）与管理端 `/portal/admin` 职责分离；静态图路径 `/portal/notice|activity|club/**` 免登录可访问。

---

## 2. 存储路径约定

利用编号中已嵌入的 ClubCategory 2位前缀做目录分层，所有文件存入 `static/portal/`（配置项 `portal.upload-dir`）下，由 Spring 静态资源映射直接提供 HTTP 访问。

### 2.1 路径规则

```
static/portal/notice/{前缀}/{uuid}.{ext}
static/portal/activity/{前缀}/{uuid}.{ext}
static/portal/club/{前缀}/{uuid}.{ext}
```

### 2.2 前缀提取

| 上传类型 | 前缀来源 | 方式 |
|---|---|---|
| 通知封面 | noticeNo | `noticeNo.substring(0, 2)`（编号前2位即 ClubCategory 缩写） |
| 活动封面 | activityNo | `activityNo.substring(0, 2)` |
| 社团 Logo | club.category | `ClubCategory.prefixOf(club.getCategory())` |

### 2.3 示例

```
通知编号 WH20260718143082739 → static/portal/notice/WH/a1b2c3d4.png
活动编号 XS20260718143091928 → static/portal/activity/XS/e5f6g7h8.jpg
社团 文化体育类              → static/portal/club/WH/i9j0k1l2.png
```

HTTP 访问路径：`/Mass_Test/portal/notice/WH/a1b2c3d4.png`（含 context-path）

## 3. 文件名校验

| 校验项 | 规则 |
|---|---|
| 文件类型 | 仅允许 `image/jpeg`、`image/png`、`image/gif`、`image/webp`（扩展名同步校验） |
| 文件大小 | 业务上限 **5MB**（Service 校验）；Spring multipart 全局上限见 `application.yml` |
| 空文件 | 拒绝（`file.isEmpty()` → 抛 BAD_REQUEST） |

---

## 4. 文件命名

`{UUID}.{原始扩展名}`，例 `a1b2c3d4e5f67890abcdef1234567890.png`。

- 使用 UUID 避免重名和中文文件名问题
- 扩展名从原始文件名提取（`FilenameUtils.getExtension`），统一转小写；`jpeg` 归一为 `jpg`

## 5. Service 层

### 5.1 PortalFileService / PortalFileServiceImpl（已实现）

| 步骤 | 逻辑 |
|---|---|
| 1 | 校验文件类型、大小、非空 |
| 2 | 解析前缀：通知/活动从编号截取，社团从 club.category → ClubCategory.prefixOf |
| 3 | 生成文件名：`UUID + 扩展名` |
| 4 | 构建目标路径：`static/portal/{type}/{prefix}/{filename}` |
| 5 | 创建目录（`mkdirs`），写入新文件 |
| 6 | 更新对应实体字段（`coverImage` / `logo`） |
| 7 | 删除旧图（如有，失败仅 log 不抛异常） |
| 8 | 返回相对路径 `/portal/{type}/{prefix}/{filename}` |

### 5.2 实体字段更新

| 上传类型 | 实体 | 更新字段 |
|---|---|---|
| 通知封面 | `NoticeInfo` | `coverImage` |
| 活动封面 | `ActivityApply` | `coverImage` |
| 社团 Logo | `SysClub` | `logo` |

## 6. 返回格式

```json
{
  "code": 0,
  "msg": "success",
  "data": "/portal/notice/WH/a1b2c3d4e5f67890.png"
}
```

前端直接拼接域名（含 context-path）即可预览。

## 7. 错误处理

| 场景 | 错误 |
|---|---|
| noticeNo 不存在 | `NOTICE_NOT_FOUND`（**6401**） |
| activityNo 不存在 | `ACT_APPLY_NOT_FOUND`（**7301**） |
| clubId 不存在 | `CLUB_NOT_FOUND`（**7202**） |
| 文件类型不合法 | `BAD_REQUEST` +「仅支持 jpg/png/gif/webp 格式」 |
| 文件为空 | `BAD_REQUEST` +「文件不能为空」 |
| 超过 5MB | `BAD_REQUEST` +「图片大小不能超过5MB」 |
| Spring multipart 超限 | `MaxUploadSizeExceededException` → 全局处理「上传文件过大」 |
