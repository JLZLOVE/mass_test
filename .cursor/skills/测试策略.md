## 2. 边界值分析（按 Service）

### 2.1 ActivitySignServiceImpl — 签到（风险最高）

| 方法 | 边界值 | 等价类 |
|------|--------|--------|
| **signIn** | `signStartTime` ±1秒, `signEndTime` ±1秒 | 提前/准时/迟到/超时 |
| | 已签到(重复), 未签到 | 正常签到/重复签到 |
| | 时间冲突(同用户同时段其他活动) | 有冲突/无冲突 |
| | `activity.status` ≠ 4 | 非已通过活动 |
| **signOut** | `enableCheckout` = 0/1 | 未启用签退/启用 |
| | 未签到先签退 | 非法操作 |
| | `signEndTime` ±1秒 | 签退窗口边界 |
| **reSign** | 补签申请→审批→执行 | 完整状态机 |
| | 社长权限校验 | 非社长申请补签 |
| **adminSign** | 管理员代签 | 权限校验 |
| | 被代签用户状态(正常/禁用) | 无效用户 |
| **stats** | 0人签到, 1人, N人 | 边界统计 |
