\# 社团综合管理平台 · 工作台（Dashboard）企业级交互设计规范 v1.0



> 文档性质：\*\*纯设计规范与业务逻辑说明书\*\*  

> 适用范围：`club-admin-frontend` 登录后首页（工作台）  

> 设计原则：\*\*权限驱动、数据隔离、克制优雅、零硬编码\*\*



\---



\## 1. 设计定位与目标



工作台是用户登录后的“第一落点”，其核心价值并非信息堆砌，而是充当 \*\*“业务导航中枢”\*\* 与 \*\*“待办聚合器”\*\*。



\- \*\*对管理者（Level 0/1）\*\*：提供全局俯瞰视角与高效审批入口。

\- \*\*对执行者（Level 2/3）\*\*：清晰呈现自身发起任务的流转状态与执行反馈。

\- \*\*对参与者（Level 4）\*\*：聚焦个人参与轨迹与平台动态通知。



工作台不承载任何具体的新增、编辑或删除操作（禁止硬编码操作按钮），所有功能执行必须下沉至二级子页面，工作台仅提供 \*\*“带语义参数的导航跳转”\*\*。



\---



\## 2. 色彩系统（Design Tokens）



遵循 \*\*“蓝主金辅、克制点缀”\*\* 的视觉基调，确保长时间办公的视觉舒适度与功能引导的明确性。



| 色彩属性 | 色值（HEX/RGBA） | 使用场景约束 | 全局占比 |

| :--- | :--- | :--- | :--- |

| \*\*主品牌蓝（深）\*\* | `#1E3A5F` | 顶部导航栏底色、一级标题、主要按钮默认态 | 约 60% |

| \*\*交互蓝（明）\*\* | `#4A90D9` | 可点击文字、悬浮态、进度条主色、激活指示器 | 约 25% |

| \*\*基础背景\*\* | `#F0F4F8` | 全局页面底色（灰蓝调，降低视觉疲劳） | — |

| \*\*香槟金（强调）\*\* | `#D4AF37` | \*\*仅用于\*\*：关键数值字体、超管/社长头像边框、紧急预警装饰条 | \*\*≤ 5%\*\* |

| \*\*浅金衬托\*\* | `#FDF1D6` | 待办卡片或高数值统计卡的轻量背景衬托 | — |

| \*\*功能红\*\* | `#E74C3C` | 超期（>7天）预警文字、退出登录按钮文字 | — |

| \*\*毛玻璃通用值\*\* | `rgba(255,255,255,0.75)` + `backdrop-filter: blur(12px)` | 统计卡片、头像下拉菜单浮层 | — |



> \*\*设计铁律\*\*：严禁将香槟金用作背景大色块或大面积边框，仅允许作为“视线聚焦点”出现。

\## 3. 页面层级与区域权重（五区布局）



工作台采用 \*\*“上下结构 + 左右分栏”\*\* 的混合布局，严格遵循视觉动线（从左至右、从上至下）。



| 区域编号 | 区域名称 | 高度/宽度占比 | 内容定义 |

| :--- | :--- | :--- | :--- |

| \*\*I区\*\* | \*\*全局导航栏\*\* | 固定高 64px | 承载系统 Logo、全局搜索（占位）、通知铃铛、头像用户菜单。 |

| \*\*II区\*\* | \*\*系统预警条\*\* | 高 40px，\*\*条件渲染\*\* | 仅当后端超时任务（>7天）存在时出现。展示严重等级警告，附带“查看详情”文字链。 |

| \*\*III区\*\* | \*\*核心指标卡\*\* | 高度自适应，约占视口 30% | 展示 4 个（或动态减少）关键 KPI 卡片。采用 A 方案：宽屏平铺，数字巨大化。 |

| \*\*IV区\*\* | \*\*主内容工作台\*\* | 占据剩余全部高度，\*\*左 65% / 右 35%\*\* | 左侧：动态任务流/申请进度列表；右侧：通知聚合与个人画像。 |

| \*\*V区\*\* | \*\*悬浮动作按钮\*\* | 固定于右下角，不与任何元素重叠 | 业务快捷入口（FAB）。仅对具备创建权限的角色可见。 |

\## 4. 头部系统（含阿里云风格用户菜单）



\### 4.1 全局导航栏（I区）构成

\- \*\*左侧\*\*：Logo + 系统名称“社团综合管理平台”。

\- \*\*中部\*\*：全局搜索框（当前阶段为 UI 占位，交互语义为“全局模糊搜索”，待后端聚合接口就绪后激活）。

\- \*\*右侧（从左至右）\*\*：

&#x20; 1. \*\*通知铃铛\*\*：带未读数量徽章。点击弹出下拉抽屉（展示最近 5 条未读），底部固定“查看全部消息”入口。

&#x20; 2. \*\*用户头像与名称\*\*：触发阿里云风格悬浮菜单。



\### 4.2 头像下拉菜单（核心交互规格）

\*\*触发方式\*\*：单击用户头像区域。  

\*\*浮层规格\*\*：宽度 280px，圆角 12px，带有 2px 浅灰边框与深阴影。



\*\*浮层内部三段式结构\*\*：

1\. \*\*顶部身份区（背景：主品牌蓝 `#1E3A5F`，高度 72px）\*\*：

&#x20;  - 左：圆形头像（直径 40px），白色描边。

&#x20;  - 右：用户真实姓名（白色加粗）；下一行显示 \*\*当前生效角色名称\*\*（如“超级管理员”）。

&#x20;  - \*\*特殊标识\*\*：若当前用户有效等级为 `0`（超管）或 `2`（社长），头像外圈自动叠加 2px 香槟金光环。

&#x20;  - 极客信息：右下角以 10px 灰色字体显示 `Level: X`（仅供运维参考，不隐藏）。



2\. \*\*功能菜单区（悬停交互：背景变 `#F0F7FF`）\*\*：

&#x20;  - 「个人中心」→ 跳转至用户资料编辑页。

&#x20;  - 「安全设置」→ 跳转至密码修改与多因素认证绑定页。

&#x20;  - 「消息中心」→ 跳转至全量通知列表页，右侧携带红点提示（未读总数）。



3\. \*\*底部退出区（分割线以上）\*\*：

&#x20;  - 「退出登录」：文字色为功能红。点击后唤起系统确认框：“确认退出当前账号？”，确认后执行登出并清空本地会话。

\## 5. 核心指标卡（III区）· 动态渲染规范



\### 5.1 卡片通用设计

\- 背景：毛玻璃效果。

\- 内边距：统一 24px。

\- 数字字号：`32px` 或 `36px`，\*\*加粗\*\*，采用香槟金 `#D4AF37`。

\- 环比变化：以小字号 + 箭头（↑/↓）展示，数据来源于后端聚合计算。

\- \*\*交互行为\*\*：卡片整体具备点击态（`cursor: pointer`），点击后携带当前业务状态参数（如 `status=pending`）路由跳转至对应的管理列表页。



\### 5.2 千人千面数据映射矩阵

系统依据当前会话的 \*\*有效等级（effectiveLevel）\*\* 动态决定展示哪 4 个指标。若某个角色层级本身不具备第 4 个指标的业务含义，则遵循 \*\*3B 规则（直接隐藏该卡片，剩余卡片自动均分父容器宽度）\*\*，严禁展示“无权限”占位符。



| 有效等级 | 卡片一（宏观/统计） | 卡片二（待办/流程） | 卡片三（执行/参与） | 卡片四（专属/预警） |

| :--- | :--- | :--- | :--- | :--- |

| \*\*0 - 超级管理员\*\* | 平台总用户数 | 全平台待审批总量 | 当前活跃社团数 | 系统连续运行天数 |

| \*\*1 - 院长/指导老师\*\* | 管辖范围内学生总数 | 待本人审批的任务数 | 本周管辖内活动总数 | 超时未处理预警数 |

| \*\*2 - 社团社长\*\* | 本社团总成员数 | 本人发起的进行中申请数 | 今日活动签到率 | 待提交的活动总结数 |

| \*\*3 - 部门部长\*\* | 本部门成员数 | 本人草稿/待提交申请数 | 近期部门参与活动数 | 部门内待办协作任务数 |

| \*\*4 - 普通学生\*\* | 本人参与的活动总数 | 本人的累计签到次数 | 平台未读通知数 | \*\*（该卡片隐藏，三列均分）\*\* |

\## 6. 主内容工作台（IV区）· 左右分栏规范



\### 6.1 左侧区域（宽度 65%）：任务/申请动态流

此区域的数据源依据用户身份自动切换，严禁混排。



\- \*\*当用户为 Level 0 或 Level 1（审批方）\*\*：

&#x20; - 数据源：全平台或管辖范围内的 \*\*待审批任务列表\*\*（`approve\_status = 待审批`）。

&#x20; - 列表项展示：活动名称、发起人姓名、当前流转步骤（如“第2步/指导老师”）。

&#x20; - \*\*状态标识\*\*：停留超过 3 天的任务，左侧边缘渲染 3px 香槟金竖条；超过 7 天的任务，左侧边缘渲染 3px 功能红竖条，且背景层叠加极淡红色脉冲动画。

&#x20; - 交互：点击列表项任意位置，路由跳转至该业务的审批详情页（对应 2A 规则：必须进详情页操作）。



\- \*\*当用户为 Level 2 或 Level 3（发起方/执行方）\*\*：

&#x20; - 数据源：\*\*当前用户自身提交的申请进度列表\*\*。

&#x20; - 列表项展示：申请名称、当前状态（待审批/已通过/已驳回/变更中）。

&#x20; - 交互：点击进入详情页仅可查看流转日志或执行“撤回”（仅在状态为“待审批”时可用）。



\### 6.2 右侧区域（宽度 35%）：通知聚合与个人画像

\- \*\*上半部分（高度占比 60%）\*\*：

&#x20; - 标题：“最新动态 / 通知”。

&#x20; - 数据源：当前用户收件箱中最新的 3 条未读或置顶通知。

&#x20; - 条目展示：标题（超出折行）、发布时间（相对时间，如“3小时前”）。

&#x20; - 底部固定文字链：“查看全部通知” → 跳转至收件箱。



\- \*\*下半部分（高度占比 40%）\*\*：

&#x20; - 标题：“我的画像”。

&#x20; - 展示字段：当前用户头像、姓名、主要所属社团/部门。

&#x20; - \*\*多身份切换\*\*：若用户关联多个社团社长角色（即 `primaryClubId` 可切换），此处展示“切换社团”文字链，点击后刷新全局菜单树与数据权限范围。

\## 7. 异常与预警机制（II区）



\- \*\*触发条件\*\*：后端定时任务（`ActivityApprovalTimeoutTask`）检测到存在 \*\*超过 7 天\*\* 未处理的审批事项。

\- \*\*展示位置\*\*：导航栏下方、核心指标卡上方。

\- \*\*视觉风格\*\*：通栏浅红底色（`#FFF5F5`），左侧边缘压 4px 香槟金粗条。文案格式：“系统检测到 X 项活动审批严重超时（>7天），建议尽快处理。”

\- \*\*交互行为（遵循 1A 规则）\*\*：

&#x20; - 右侧提供文字链“查看详情”。

&#x20; - 点击后跳转至活动审批列表页，并自动激活“超时筛选”条件。

&#x20; - \*\*此处禁止放置“一键转交”或“批量通过”等操作按钮\*\*，所有审批动作必须下沉至二级详情页，以防页面误触导致流程紊乱。



\---



\## 8. 悬浮动作按钮（V区）· 权限边界



\- \*\*可见性规则\*\*：仅当用户有效等级 ≤ 2（即超管、院长/指导老师、社长）时，右下角渲染该组件。

\- \*\*主按钮设计\*\*：直径 56px 圆形，采用香槟金渐变背景，悬停时轻微放大（Scale 1.05）。

\- \*\*展开菜单项（依据等级动态过滤）\*\*：

&#x20; - 「发起活动」：对 Level ≤ 2 可见。

&#x20; - 「发布通知」：仅对 Level ≤ 1 可见（普通社长无全平台通知权限）。

&#x20; - 「签到管理」：对 Level ≤ 2 可见，跳转至今日待签到活动列表。

\- \*\*交互闭环\*\*：点击菜单项后，FAB 自动收起，页面执行路由跳转。

\## 9. 页面交互闭环与数据刷新策略（非功能需求）



\- \*\*首次加载\*\*：页面进入时并行请求 Dashboard 聚合数据与未读消息数，骨架屏（Skeleton Screen）优先渲染卡片占位，避免白屏。

\- \*\*数据保鲜\*\*：工作台不启用自动轮询（避免冗余请求占用带宽）。仅在以下场景触发手动刷新：

&#x20; 1. 用户从二级子页面（如审批详情页）通过浏览器后退或面包屑返回工作台时，触发 `onActivated`（Keep-Alive）局部刷新。

&#x20; 2. 用户点击顶部导航栏 \*\*Logo\*\* 或 \*\*“工作台”\*\* 面包屑时，强制刷新当前 Dashboard 数据。

\- \*\*跳转携带语义\*\*：所有从工作台发出的跳转（卡片点击、预警条点击、任务列表点击）必须在路由 `query` 参数中携带\*\*明确的筛选条件\*\*（如 `status=pending` 或 `overdue=true`），保证目标页面与工作台意图无缝衔接，杜绝“跳转后需用户二次筛选”的割裂体验。

\## 10. 设计验收标准（Checklist）



\- \[x] 色彩系统符合蓝金配比，金色占比未超过 5% 阈值。

\- \[x] 所有数据展示均依赖后端接口字段，前端未写死任何 Mock 假数据或硬编码角色 ID。

\- \[x] 千人千面逻辑基于 `effectiveLevel` 驱动，卡片数量支持动态增删（3B 规则）。

\- \[x] 工作台不包含任何表单提交或数据变更按钮（删除/编辑/通过/驳回），所有写操作均在二级页闭环。

\- \[x] 预警条仅作导航跳转，不携带业务处置功能（1A 规则）。

\- \[x] 头像菜单层级清晰，包含极客向信息（Level 展示）但不影响常规用户体验。

\- \[x] 悬浮按钮（FAB）依据权限动态显隐，且不遮挡任何页面核心内容。
