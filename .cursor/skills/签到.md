## 签到模块完整逻辑

> 与代码 / `PROJECT_OVERVIEW.md` 同步（2026-07-18）：对外统一 **`activityNo`**，配置与签退含时间窗校验。

### 一、签到配置

每个活动单独配置签到规则：

**入参**：

| 字段 | 说明 |
|---|---|
| **activityNo** | 活动编号（对外标识，**禁止传 activityId**） |
| signMode | 1定位 / 2扫码 / 3两者 |
| signStartTime / signEndTime | 签到窗口；须 `start < end`（`@StartBeforeEnd`） |
| | **start ≥ 当前时间**（6516）；**end ≤ start + 7 天**（6517） |
| signRadius | 默认 100 米 |
| enableCheckout | 是否启用签退 |
| centerLatitude / centerLongitude | 定位中心（可选） |

**持久化**：`activity_sign_config.activity_id`（内部）+ **`activity_no`**（对外列）；响应 `@JsonIgnore activityId`。

**权限**：活动负责人（发起人）可配置，指导老师及以上可修改。

### 二、签到方式

| 方式 | 适用场景 | 说明 |
|---|---|---|
| 扫码签到 | 线下活动 | 生成 qrToken，扫码即签到 |
| 定位签到 | 线上/线下 | Haversine 半径校验 |
| 两者都支持 | 混合场景 | 用户任选 |

### 三、签到执行

1. 窗口期内 → 允许签到  
2. 定位半径 / 扫码令牌校验  
3. 写入 `activity_sign`；迟到：签到时间 > 活动开始时间  
4. 同一用户同一活动去重  
5. 同时段其他活动冲突 → **6511**

**手动签到**：Body 传 **`username`**（非 userId）。

### 四、签退

1. 须已启用签退  
2. **签退时间 ≥ 签到时间**（6518）且 **≤ 签到 + 7 天**（6519）  
3. 签退 < 活动结束 → 早退  
4. 活动结束后未签退 → `ActivitySignCheckoutTask` 自动签退（钳制在签到～签到+7天）

### 五、补签审批

**发起人**：社长；入参 **`activityNo`** + 补签人 **`username`** + 原因类型/说明/附件。  
活动结束 +1 天内可申请。院级指导老师 / 校级指导老师→学院书记。

### 六、签到统计

应签 / 已签 / 签到率 / 迟到 / 早退 / 半小时分布；VO 暴露 **`activityNo`**。

## 接口清单

| 接口 | 方法 | 路径 | 说明 |
|---|---|---|---|
| 配置签到 | POST | `/activity-sign/config` | Body 含 **activityNo** |
| 更新配置 | PUT | `/activity-sign/config` | 同上 |
| 查询配置 | GET | `/activity-sign/config?activityNo=` | 响应含 activityNo |
| 签到 | POST | `/activity-sign/sign?activityNo=` | 定位/扫码 |
| 手动签到 | POST | `/activity-sign/admin/sign?activityNo=` | Body: username |
| 签退 | POST | `/activity-sign/checkout?activityNo=` | |
| 申请补签 | POST | `/activity-sign/apply?activityNo=` | |
| 补签审批 | POST | `/activity-sign/approve/{applyId}` | |
| 签到统计 | GET | `/activity-sign/stats?activityNo=` | |
| 签到明细 | GET | `/activity-sign/list?activityNo=` | |

**错误码**：6501~6519（见 `Erroconfig.md` / `PROJECT_OVERVIEW` §9.7）

## 编码约束

| 项 | 要求 |
|---|---|
| SQL | MyBatis-Plus Lambda |
| 对外标识 | **activityNo** / **username**，不暴露内部 id |
| 事务 | 签到、补签审批 `@Transactional` |
| 去重 | 同一用户同一活动一条记录 |
