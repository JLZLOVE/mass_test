## 签到模块完整逻辑

### 一、签到配置

每个活动单独配置签到规则：

**入参**：活动ID、签到方式（1定位/2扫码/3两者都支持）、签到开始时间、签到结束时间、签到半径（默认100米，可调）、是否启用签退

**默认值**：不配置时默认不开启签到

**权限**：活动负责人（发起人）可配置，指导老师及以上可修改

### 二、签到方式

| 方式       | 适用场景      | 说明                               |
| :--------- | :------------ | :--------------------------------- |
| 扫码签到   | 线下活动      | 生成二维码或动态码，用户扫了即签到 |
| 定位签到   | 线上/线下活动 | 在半径范围内自动签到               |
| 两者都支持 | 混合场景      | 用户任选一种                       |

**扫码签到**：活动开始前系统生成二维码，活动当天展示给参与人扫码。扫码后直接记录签到时间，无需额外审批。

### 三、签到执行

用户点击签到 → 系统判断：

1. 签到窗口期内 → 允许签到
2. 是否在签到点半径内（定位）或是否扫码（扫码）
3. 通过 → 写入 `activity_sign`，记录签到方式、时间、地点
4. 迟到判定：签到时间 > 活动开始时间 → 标记迟到
5. 一人多次签到 → 去重，只记录首次有效签到

**手动签到**：管理员后台可手动标记某人已签到（用于网络故障等），记录操作人

### 四、签退

由活动负责人决定是否启用签退功能。启用时：

1. 用户签到后可点击签退
2. 签退时间 < 活动结束时间 → 标记早退
3. 签退时间 ≥ 活动结束时间 → 正常签退
4. 活动结束后未签退 → 系统自动签退，不标记早退

### 五、补签审批

**发起人**：社长（活动负责人）为参与人发起补签申请

**入参**：活动ID、补签人ID、补签原因类型（1活动方原因/2个人原因/3特殊情况）、具体说明、附件（证明材料）

**审批链**：

- 院级活动（`activity_level=1`）：指导老师审批
- 校级活动（`activity_level=0`）：指导老师 → 院系管理员（`sys_college.dean_id`）

**审批规则**：

- 活动方原因 → 通常同意
- 个人原因 → 非重大原因不同意
- 特殊情况 → 酌情处理

**补签期限**：活动当天内可申请，超过1天不可申请

**审批动作**：通过后系统写入签到记录，`sign_type=3`（补签），记录审批人

### 六、签到统计

| 统计项       | 计算方式                                                     |
| :----------- | :----------------------------------------------------------- |
| 应签人数     | `activity_apply.expected_people`（预约人数，如果无预约则用实际签到+未签到） |
| 已签到人数   | 定位签到 + 扫码签到 + 手动签到 + 补签通过（去重）            |
| 未签到人数   | 应签人数 - 已签到人数                                        |
| 签到率       | 已签到人数 / 应签人数 × 100%                                 |
| 迟到人数     | `is_late=true` 的人数                                        |
| 早退人数     | `is_early_leave=true` 的人数                                 |
| 签到时间分布 | 每半小时签到人数统计                                         |
| 签到明细     | 姓名、签到方式、签到时间、签到地点、是否迟到/早退            |

### 七、异常处理

- 定位不准确：用户可手动修改签到位置（需审批或管理员确认）
- 扫码失效：管理员手动签到作为兜底
- 签到时间冲突：同一人在同一时间段不能签到两个不同的活动（防止串场）

## 接口清单

| 接口     | 方法 | 路径                                     | 权限                | 说明                               |
| :------- | :--- | :--------------------------------------- | :------------------ | :--------------------------------- |
| 配置签到 | POST | `/activity-sign/config`                  | 活动负责人          | 设置签到方式、时间、半径、签退开关 |
| 查询配置 | GET  | `/activity-sign/config/{activityId}`     | 参与人              | 查看签到点信息                     |
| 更新配置 | PUT  | `/activity-sign/config`                  | 活动负责人/指导老师 | 修改签到规则                       |
| 签到     | POST | `/activity-sign/sign/{activityId}`       | 参与人              | 扫码/定位签到                      |
| 手动签到 | POST | `/activity-sign/admin/sign/{activityId}` | 管理员              | 后台标记签到                       |
| 签退     | POST | `/activity-sign/checkout/{activityId}`   | 已签到参与人        | 签退                               |
| 申请补签 | POST | `/activity-sign/apply/{activityId}`      | 社长                | 发起补签申请                       |
| 补签审批 | POST | `/activity-sign/approve/{applyId}`       | 指导老师/院系管理员 | 通过/驳回                          |
| 签到统计 | GET  | `/activity-sign/stats/{activityId}`      | 活动负责人/指导老师 | 返回统计报表                       |
| 签到列表 | GET  | `/activity-sign/list/{activityId}`       | 活动负责人/指导老师 | 分页查看签到明细                   |

**已移除的旧接口**：`listActivitySign_F/B`、`detailActivitySign_F/B`、`add_B`、`updateActivitySign_B`、`deleteActivitySign_B`、`query`

## 编码约束

| 项   | 要求                                             |
| :--- | :----------------------------------------------- |
| SQL  | MyBatis-Plus Lambda，禁止字符串拼接              |
| 权限 | `LevelBasedAccess`、`SecurityUtils`、现有 Helper |
| 分页 | `MPUtil.getPage`                                 |
| 日志 | `@Slf4j`，不用 `System.out`                      |
| 事务 | 签到、补签审批必须加 `@Transactional`            |
| 去重 | 同一用户同一活动只能有一条签到记录               |
