# Cursor 项目分析指令

> **目标**：基于当前后端项目源码、配置和 SQL 文件，自动生成一份结构化的项目说明文档 `PROJECT_OVERVIEW.md`，用于后续重构和团队沟通。  
> **执行前提**：请确保 Cursor 当前工作目录为项目根目录，且已加载所有相关文件（.java, .xml, .sql, .yml 等）。

---

## 任务总览

1. 收集项目基本信息（技术栈、依赖、启动类、配置）
2. 解析数据库结构（表、字段、关系）
3. 扫描代码包结构，梳理分层职责
4. 提取所有 Controller 接口，生成接口清单
5. 分析认证与授权机制（JWT + Redis）
6. 汇总公共组件（统一响应、异常处理、分页、缓存）
7. 生成最终 `PROJECT_OVERVIEW.md` 文档

## 执行步骤

### 1. 项目基本信息收集

- 读取 **`pom.xml`**（或 `build.gradle`），记录：
  - `groupId`、`artifactId`、`version`、`name`
  - Spring Boot 版本、主要依赖（MyBatis-Plus、Knife4j、JWT、Redis、Lombok 等）
- 查找主启动类：扫描所有包含 `@SpringBootApplication` 的类，记录其包路径。
- 读取 **配置文件**（`src/main/resources/application.yml），提取：
 
  - JWT 密钥（`jwt.secret` 或类似属性）
  - 日志级别（`logging.level`）
  - 其他重要自定义配置（如文件上传路径、跨域设置等）

> 若某些文件不存在或配置项缺失，请在输出中备注“未找到”。
 - 数据库连接（URL、驱动、用户名）: root、mysql、123456
  - Redis 连接（host、port、password）：、root 、6379、123456

### 2. 数据库结构分析

- 扫描项目D:\java\Mass_Test\mysql下的所有 `mass_test1.sql.sql` 文件。
- 解析 `CREATE TABLE` 语句，提取：

  - 主键（`PRIMARY KEY`）、索引（`INDEX/KEY`）、外键约束（`FOREIGN KEY`）

- 若存在数据字典（如枚举值说明），一并整理。

### 3. 代码结构扫描

#### 3.1 包结构概览
- 列出 `src/main/java` 下的所有一级包，如：`controller`、`service`、`service.impl`、`mapper`、`entity`、`dto`、`config`、`filter`、`interceptor`、`utils`、`model`、`exception`。
- 简要描述每个包的职责（例如：`controller` 负责接收 HTTP 请求，`service` 定义业务接口，`service.impl` 实现业务逻辑）。

#### 3.2 Controller 层（接口清单）
- 找到所有标注 `@RestController` 的类。
- 对于每个 Controller：
  - 记录其 `@RequestMapping` 的根路径。
  - 遍历所有方法（`@GetMapping`、`@PostMapping`、`@PutMapping`、`@DeleteMapping` 等），提取：
    - **完整 URL**：根路径 + 方法路径
    - **HTTP 方法**
    - **接口描述**：优先读取 `@Operation` 的 `summary` 和 `description`，若无则读取 JavaDoc 第一行。
    - **访问控制**：若方法或类上标注了 `@IgnoreAuth`，标记为“公开接口”；否则为“需认证”。
    - **请求参数**：
      - `@RequestParam`（参数名、类型、是否必填、默认值）
      - `@PathVariable`（参数名、类型）
      - `@RequestBody`（对应的 DTO/Entity 类名及字段含义）
    - **响应格式**：统一为 `R` 对象（包含 `code`、`msg`、`data` 字段），记录 `data` 中可能的数据类型。
  - 按 Controller 类分组，生成接口清单表格（见下文输出格式）。

#### 3.3 Service 层与业务逻辑
- 扫描所有 `@Service` 类（包括实现类），记录：
  - 类名及其职责（如 `ActivityApplyService` 负责活动申请业务）
  - 核心方法（尤其是事务方法 `@Transactional`、缓存方法 `@Cacheable`、异步方法 `@Async`）
  - 主要依赖（其他 Service、Mapper、第三方客户端）
- 梳理主要业务模块：例如“活动申请与审批”、“公告管理”、“用户管理”、“权限控制”等，并简要描述每个模块的核心功能。

#### 3.4 Mapper 层与数据访问
- 扫描所有继承自 `BaseMapper<T>` 的接口（标注 `@Mapper`），记录：
  - 对应的实体类（`T`）及数据库表名
  - 自定义查询方法（非 MyBatis-Plus 内置 CRUD），提取方法名和可能的 SQL 注解（`@Select`、`@Insert` 等）
- 关联到第 2 步中的数据库表。

#### 3.5 Entity 与 DTO
- 列出所有标注 `@TableName` 的实体类，记录：
  - 类名 → 数据库表名
  - 每个字段对应的数据库列（`@TableField`）、类型、是否为主键（`@TableId`）
- 列出专用 DTO 类（如 `RegisterDTO`、`LoginDTO`、`QueryDTO`），记录其用途（通常用于请求参数或数据传输）。

### 4. 核心功能与安全机制

#### 4.1 认证与授权（JWT 链路）
- 定位 JWT 相关类：
  - `JwtUtil`（签发 Token、验证 Token、解析 Claims）
  - `JwtFilter`（继承 `OncePerRequestFilter`，拦截请求，验证 Token，设置 SecurityContext）
  - 配置类（如 `SecurityConfig` 或 `WebSecurityConfig`，声明 `AuthenticationManager`、密码编码器等）
- 分析认证流程：
  1. 客户端调用登录接口（如 `/login/allocation`），提交用户名密码。
  2. 接口调用 `AuthenticationManager.authenticate()`，触发 `UserDetailServiceImpl.loadUserByUsername()` 加载用户信息及权限。
  3. 验证成功后，`JwtUtil` 生成 Token（subject=`username`），并将 **`LoginUserDetails.CacheSnapshot`** 写入 Redis（Key：**`user:v2:{username}`**，TTL 1h）。
  4. 返回 Token 与 username 给客户端。
  5. 注销：`POST /login/logout` → `UserCacheHelper.evictByUsername`；后续同 Token 因 Redis 缺失由 JwtFilter 返回 401（**不回源 DB**）。
- 授权流程：
  - 除 `@IgnoreAuth` / `security.ignore.urls`（含门户静态图 `/portal/notice|activity|club/**`）外，请求需携带 `Token` 或 `Bearer`。
  - `JwtFilter` 校验 Token → 读 Redis 会话 → 注入 `LoginUserDetails`。

#### 4.2 权限与数据权限
- 涉及表：`SysUserRole`（用户-角色）、`SysRoleMenu`（角色-菜单）、`SysDataPermission`（数据权限规则）。
- 说明如何控制菜单可见性（根据角色查询菜单）和业务数据范围（如只查看本人或本部门数据）。
- 提及工具类（如 `MPUtil`）中的 `likeOrEq`、`between`、`sort` 等方法如何配合 MyBatis-Plus 的 `QueryWrapper` 实现动态过滤。

#### 4.3 公共组件
- **统一响应**：`R` 类（包含静态方法 `ok()`、`error()`，字段 `code`、`msg`、`data`）。
- **全局异常处理**：`GlobalExceptionHandler`（标注 `@RestControllerAdvice`），处理业务异常、参数校验异常、系统异常，返回标准 `R` 格式。
- **分页工具**：`MPUtil.getPage()` 配合 MyBatis-Plus 分页插件（`PaginationInterceptor` 或 `MybatisPlusInterceptor`），解析分页参数（`page`、`limit`、`sort` 等）。
- **Redis 缓存**：除了存储权限，还可能用于其他业务缓存（如字典、会话等），记录使用的缓存注解或直接操作 Redis 的工具类。
